# Evacuation Command

Evacuation Command 是一个浏览器端 2D 网页游戏项目。当前版本在基础战斗上加入了“获取情报并撤离”的完整任务闭环。所有游戏对象均通过 Phaser Graphics 程序化绘制，不依赖外部美术、音乐或字体素材。

## 当前版本功能

- 页面标题为 `Evacuation Command`。
- 游戏画面使用 960×540 的设计分辨率，并通过 Phaser Scale FIT 随浏览器窗口等比缩放、居中显示。
- 深色背景和简易地图边界、网格均由 Phaser Graphics 绘制。
- 蓝色玩家角色、黄色子弹和红色近战敌人均由 Phaser Graphics 绘制。
- 支持 `W`、`A`、`S`、`D` 控制玩家移动，玩家移动会被限制在地图边界内。
- 鼠标位置决定玩家瞄准方向，鼠标左键射击，射击具有冷却时间。
- 子弹沿瞄准方向直线飞行，命中敌人会造成伤害，离开地图后自动销毁。
- 游戏开始时生成 5 个近战敌人，敌人不会出生在玩家附近。
- 敌人在检测范围内追击玩家，接近玩家后按固定间隔造成伤害。
- 玩家生命值为 100，受伤时会红色闪烁；敌人被命中会闪烁，死亡时有简单消失反馈。
- 地图中部会生成黄色情报物品；接触后自动拾取，并激活右上角撤离区域。
- 获取情报后每 5～7 秒会尝试补充一名近战敌人，同时存活敌人最多 8 名。
- 左上角显示玩家生命值、击杀数、存活敌人数、任务目标、情报状态、180 秒倒计时和撤离进度。
- 玩家必须取得情报，再在右上角撤离区域内连续坚守 3 秒才能成功；离开区域会清空进度。
- 消灭初始敌人不会直接胜利。玩家生命值归零或任务倒计时结束会失败。
- 结算界面显示中文结果和本局击杀数，按 `R` 可以重新开始完整场景。

## 操作说明

- `W` / `A` / `S` / `D`：移动玩家。
- 移动鼠标：控制瞄准方向。
- 鼠标左键：射击。
- `R`：重新开始当前场景。

## 核心任务循环

1. 从地图左下区域进入，按需对抗初始 5 名近战敌人。
2. 前往地图中部接触黄色菱形情报。
3. 情报取得后，前往右上角变为蓝绿色的撤离点，并应对周期性增援。
4. 在撤离点内连续停留 3 秒完成撤离；离开撤离点后进度归零。

## 成功与失败条件

- **成功**：已取得情报、进入激活的撤离区域，并在存活状态下连续坚守 3 秒。
- **失败**：玩家生命值归零，或 180 秒任务倒计时耗尽。
- 成功或失败后，战斗任务与增援停止；按 `R` 会完整重置情报、撤离点、计时、击杀数和敌人。

## 当前项目结构

- `src/entities/`：玩家、近战敌人与子弹实体。
- `src/systems/CombatSystem.ts`：射击、伤害、敌人及增援管理。
- `src/systems/MissionSystem.ts`：情报、撤离、任务倒计时与任务状态。
- `src/systems/Hud.ts`：战斗与任务状态显示、结算界面。
- `src/scenes/MainScene.ts`：组合实体和系统，协调场景生命周期。
- `src/config/` 与 `src/types/`：共享配置和严格类型定义。

## 已知问题

- 当前只有一种直接追击玩家的近战敌人，不包含复杂寻路或敌人之间的碰撞避让。
- Phaser 核心会使生产构建的 JavaScript 包体较大，Vite 可能显示 chunk size 提示，但不影响构建和运行。

## 环境要求

- Node.js 20.19+ 或 22.12+
- npm 10+

## 安装依赖

```bash
npm install
```

在 CI 或需要严格复现锁文件依赖时，也可以使用：

```bash
npm ci
```

## 开发运行

```bash
npm run dev
```

默认开发服务器地址为 <http://localhost:5173>。

## 生产构建

```bash
npm run build
```

构建产物会输出到 `dist/` 目录。

## 本地预览生产构建

```bash
npm run preview
```

## 核心依赖

- Phaser：浏览器端 2D 游戏框架。
- TypeScript：静态类型检查和开发语言。
- Vite：开发服务器和生产构建工具。
